Variable Definition in C:

A variable definition means to inform the compiler where and how much to make the storage for the variable. A variable definition specifies a knowledge type and contains an inventory of 1 or more variables of that type as follows:

type variable_list;

Here, the sort must be a legitimate C data type including char, w_char, int, float, double, bool or any user-defined object, etc., and variable_list may contain one or more identifier names separated by commas. Some valid declarations are shown here:

int i, j, k; char c, ch; float f, salary; double d;

The line int i, j, k; both declares and defines the variables i, j and k; which instructs the compiler to make variables named i, j and k of type int. Variables are often initialized (assigned an initial value) in their declaration. The initializer consists of an equal sign followed by a continuing expression as follows:

type variable_name = value;

Variable declaration in C:

A variable declaration provides assurance to the compiler that there's one variable existing with the given type and name in order that the compiler proceeds with further compilation without having complete detail about the variable. A variable declaration has its meaning at the time of compilation only, compiler needs actual variable declaration at the time of linking of the program. A variable declaration is beneficial once you are using multiple files and you define your variable in one among the files, which can be available at the time of linking the program. you'll use the extern keyword to declare a variable anywhere. Though you'll declare a variable multiple times in your C program it is often defined as just one occasion during a file, a function, or a block of code.

There are two sorts of expressions in C:

  1. lvalue:

    An expression that's an lvalue may appear as either the left-hand or right-hand side of an assignment.

  2. rvalue:

    An expression that's an rvalue may appear on the right- but not the left-hand side of an assignment.

Variables are lvalues then may appear on the left-hand side of an assignment. Numeric literals are rvalues then might not be assigned and can't appear on the left-hand side. Following may be a valid statement:

But the following isn't a legitimate statement and would generate a compile-time error:
